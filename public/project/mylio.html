Mylio Pair
Overview
This is a new function designed for Mylio photo app.

I worked as Product and UI/UX Designer on this project. I collaborated with PM, UI/UX Lead, engineers from early concepts all the way down to final shipment. All wireframes, mockups, and prototypes showed here were designed, coded and created by me.

The design helped both Mylio's free-plan users and professional users reduce their unsorted photos by significant numbers in very short time.

User Problem
EXIF is the meta data generated by any digital device when we take a photo, it includes file size, taking date etc. Not every photo has (correct) EXIF data. Time-sequencing photos without (correct) EXIF data is currently an impossible task for computers. Photo apps usually fail to recognize the real date of a scanned photo. Two main types of photos on Mylio without EXIF or without correct EXIF are old family photos uploaded by regular users and film-based photos uploaded by professional photographers.
Problematic Flow
When user scans an old photo taken in 1995 and then uploads to any photo app, the system probably recognizes its date as 2015 because that is the EXIF info the scanner generates. To solve this problem, it needs human input from the user who knows the real date of the photo. However, the manual sorting process can be long, complex and tedious.
Design Goal

"Sorting should not be a task for Mylio's user, it should be a game."

"A beautiful human sorting experience that is fast, effortless, and playful."


The overall sorting experience design can be evaluated by the following points:
  • Ask user easy question
  • Intuitive user interaction
  • Minimal user physical input
  • Sorting speed
  • Sorting accuracy
  • Reversiblity & Forgiveness
  • Feedback & Responsiveness
  • Integration with current UI
  • Adaptability to all devices
    • Pair Sorting
    Instead of asking user to organize all photos using folder or tag system - which is asking user "what" and "when" questions, Mylio Pair introduces a Pair & Compare strategy to simplify the question to "which". On front-end, every time the system only presents two photos and asks user to choose which is the older (newer) one. On back-end, the system marks user's choice and keeps re-sequencing all photos and generates new pairs for user to compare. After certain times of pair comparing, all photos can be sorted into a timeline.
    Pair Exploration
    The actual Mylio Pair design was born from three pair UI frameworks. All three options were adopting the idea of "pair and compare", but they had quite different user interaction designs. By developing and testing each of them, the final design was chosen based on a synthetic performance of intuitivity, efficiency and adaptability.

    Option 1: Landscape and Tap

    Tap to choose, three buttons to go backward, forward and undo, navigating panel on the bottom.


    + Efficiency

    + Bigger view range

     

    - Gesture inconsistency

    - Unadaptability

    - More buttons

    Option 2: Swipe and Peek-view

    Tap to choose, swipe to go backward and forward, one button to undo, navigating panel on the side.


    + Efficiency

    + Intuitivity

    + Gesture consistency

    + Adaptability

    + Less buttons

     

    - Relatively smaller view range

    Option 3: Drag and Drop

    Drag either newer or older photo and drop it into its category on the side, two buttons to go backward and forward, drag previous comparing photos back to the center to undo.


    + Intuitivity

    + Options for choosing

     

    - Low efficiency

    - Unadaptability

    - More side bars

    - Smaller view range
    User Scenarios
    After selecting the Swipe and Peek-view option, a series of user interactions were designed based on different user scenarios, which include user sorting, skipping, reversing, fast navigating and positioning etc.
    Sorting Flow
    Mylio Pair works as a human-computer double-end sorting process that turns the frustrating manual task into something easier, faster and even addictive. The sorting interactions are designed to be intuitive, responsive and speedy, which creates a game-like experience.

    1. System presents one pair of photos

    2. User selects the older photo

    3. Old-photo filter effect provides feedback

    4. System generates the next pair

    Fuzzy & Forgiving
    One further problem is that old photos can sometimes make user feel fuzzy. When user can not make a decision about the current compare, Mylio is able to let user skip and move to the next pair. When user realizes she/he made a wrong pick, she/he can easily go back to the previous pairs and view the results. User can change the result simply by choosing the other photo, or tapping the undo button to remove the existing choice and just leave it fuzzy.

    1. User can swipe up to pass the current pair if she/he feels fuzzy

    2. System generates the next pair

    3. User can also swipe down to go back to the last pair

    4. Back to the last pair and showing user's choice

    5. User changes the choice by just tapping the other photo

    6. System provides feedback for the change of choice

    7. User can undo a result via the button

    8. The old photo filter effect disappears after undo

    Fast Navigation & Immersive View
    An optional navigating panel is designed to enable user to preview and control the flow of photo pairs, which provides a better efficiency for going forwards or backwards of multiple pairs.

    1. Using the side panel, user can swipe up to skip multiple pairs

    2. Or swipe down to go back to previous pairs

    3. Side bar is toggled through a "pair" button

    4. Closing the side bar, user can have full-view gaming

    A Live Prototype

    2. Mouse click on peekview to go forward(skip) and go back(modify).

    3. Mouse click on "undo" button to undo the result.
